# Deep Benchmark Matrix

## Benchmark dimensions

For every external product/framework, extract five things:
1. What it does best.
2. What product pattern we should borrow.
3. What we should avoid.
4. What data signal it suggests.
5. How Long Learning OS should surpass it.

## Khanmigo / Khan Academy

- **Best pattern**: Socratic AI tutor; nudges learners instead of directly answering.
- **Borrow**: hint-first tutoring, parent support, skill mastery, homework help guardrails.
- **Avoid**: becoming a homework chatbot without longitudinal developmental context.
- **Signals**: hint usage, correction loops, mastery checkpoints.
- **Surpass**: combine with bilingual Vietnamese-English context, parent co-learning, adaptive acceleration, ownership track, and multi-year portfolio.

## Duolingo Max

- **Best pattern**: bite-sized habit loop; roleplay; explain-my-answer feedback.
- **Borrow**: speaking roleplays, scenario-based practice, lightweight daily habit.
- **Avoid**: streak addiction and shallow language progress.
- **Signals**: speaking turns, error type, scenario fluency, return frequency.
- **Surpass**: add deep writing, domain vocabulary, bilingual transfer, and human conversation missions with parents.

## Anki / Quizlet / Q-Chat

- **Best pattern**: flashcards, spaced repetition, fast quiz generation.
- **Borrow**: spaced recall, adaptive quiz, study from notes.
- **Avoid**: memorization without understanding.
- **Signals**: recall interval, item difficulty, forgetting risk.
- **Surpass**: require explain/apply/teach-back before marking mastery.

## Brilliant

- **Best pattern**: interactive learning by doing in math, programming, data, AI, science.
- **Borrow**: bite-sized interactive problem-solving and guided discovery.
- **Avoid**: being too abstract for young children without concrete manipulatives.
- **Signals**: steps taken, conceptual checkpoints, challenge persistence.
- **Surpass**: age-safe concrete-to-abstract path and real-life family data/projects.

## Scratch / ScratchJr

- **Best pattern**: creative coding through stories, games, animations.
- **Borrow**: creation-first computing and play-based logic.
- **Avoid**: unbounded community/social exposure for young children.
- **Signals**: project completion, blocks used, debugging iterations.
- **Surpass**: integrate progression from ScratchJr -> Scratch -> Python -> AI/data apps.

## Code.org

- **Best pattern**: free K–12 computer science and AI curriculum; teacher/parent accessible.
- **Borrow**: structured CS pathway, AI foundations, hands-on projects.
- **Avoid**: treating CS as a separate subject disconnected from family projects and ownership.
- **Signals**: concept mastery, project progression, AI literacy tasks.
- **Surpass**: link CS to the child’s own tools, portfolio, finance, and data reasoning.

## iCivics

- **Best pattern**: civics learned through simulation and role-based games.
- **Borrow**: policy/government/legal roleplay.
- **Avoid**: overly U.S.-centric content without Vietnam/global adaptation.
- **Signals**: civic vocabulary, decision quality, pre/post understanding.
- **Surpass**: build Vietnam + global rule systems, family rule analogies, geopolitics, and source credibility.

## CASEL / Harvard Executive Function

- **Best pattern**: competence frameworks for self-awareness, self-management, social awareness, relationship skills, decision-making; age-based executive function practice.
- **Borrow**: SEL/executive-function maps and activities.
- **Avoid**: moralizing dashboards or surveillance psychology.
- **Signals**: reflection, planning, emotional labeling, recovery.
- **Surpass**: connect ethics to AI, money, leadership, negotiation, and pressure scenarios.

## OECD / WEF / National Academies frameworks

- **Best pattern**: future skills, transferable skills, agency, responsibility, analytical thinking, resilience, leadership, AI/data literacy.
- **Borrow**: skill taxonomy and future readiness lens.
- **Avoid**: vague “21st century skills” without measurable behaviors.
- **Signals**: behavior-based rubrics and artifacts.
- **Surpass**: turn frameworks into daily micro-actions, projects, and measurable longitudinal growth.

## Antigravity/Codex development benchmark requirement

Coding agents must not merely implement screens. They must implement learning loops:
- diagnostic -> adaptive plan -> lesson -> attempt -> feedback -> correction -> reassessment -> memory schedule -> parent summary -> portfolio artifact.

Any feature PR without data events and assessment logic should be considered incomplete.
